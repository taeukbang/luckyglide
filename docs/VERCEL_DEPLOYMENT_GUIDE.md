# Vercel 배포 상태 확인 가이드

## 📍 Vercel 대시보드 접속

### 1단계: Vercel 웹사이트 접속
1. 브라우저를 엽니다
2. 주소창에 다음을 입력하고 Enter:
   ```
   https://vercel.com
   ```
3. 로그인 (GitHub 계정으로 로그인)

### 2단계: 프로젝트 선택
1. 로그인 후 대시보드 화면이 나타납니다
2. 왼쪽 메뉴에서 **"Projects"** 또는 **"프로젝트"** 클릭
3. 프로젝트 목록에서 **"luckyglide"** 클릭

### 3단계: 배포 상태 확인
프로젝트 페이지에서 다음을 확인할 수 있습니다:

#### 배포 목록
- 화면 상단에 최근 배포 목록이 표시됩니다
- 각 배포 항목 옆에 상태가 표시됩니다:
  - ✅ **Ready** (초록색) = 배포 완료
  - 🟡 **Building** (노란색) = 배포 중
  - ❌ **Error** (빨간색) = 배포 실패

#### 배포 상세 정보
- 배포 항목을 클릭하면 상세 정보를 볼 수 있습니다:
  - 배포 시간
  - 커밋 메시지
  - 배포 URL
  - 빌드 로그

### 4단계: 최신 배포 확인
1. 가장 위에 있는 배포 항목이 최신 배포입니다
2. 커밋 메시지가 **"feat: 파트너별 실시간 MyLink 생성 기능 추가"**인지 확인
3. 상태가 **"Ready"**인지 확인

## 🔍 배포 상태별 의미

| 상태 | 의미 | 다음 단계 |
|------|------|----------|
| **Ready** | 배포 완료 | 배포된 사이트 테스트 |
| **Building** | 배포 중 | 잠시 기다린 후 새로고침 |
| **Error** | 배포 실패 | 에러 로그 확인 후 수정 |

## 🚀 배포된 사이트 테스트

배포가 완료되면 (`Ready` 상태):

1. 배포 항목을 클릭
2. **"Visit"** 또는 **"방문"** 버튼 클릭
3. 또는 직접 URL 접속:
   ```
   https://luckyglide.vercel.app/partner1
   ```
4. 예약하기 버튼 클릭 테스트
5. 개발자 도구(F12) → Console에서 MyLink 생성 확인

## ⚙️ 환경변수 확인

배포가 실패하거나 기능이 작동하지 않으면:

1. Vercel 대시보드 → 프로젝트 선택
2. 왼쪽 메뉴에서 **"Settings"** 클릭
3. **"Environment Variables"** 클릭
4. 다음 환경변수가 있는지 확인:
   - `SUPABASE_URL`
   - `SUPABASE_ANON_KEY`
   - `MRT_PARTNER_API_KEY_partner1` (각 파트너별)

## 📝 문제 해결

### 배포가 안 되거나 실패하는 경우

1. **GitHub에 푸시 확인**
   - GitHub 저장소에서 최신 커밋이 있는지 확인
   - `https://github.com/taeukbang/luckyglide` 접속

2. **Vercel과 GitHub 연결 확인**
   - Vercel 대시보드 → Settings → Git
   - GitHub 저장소가 연결되어 있는지 확인

3. **빌드 로그 확인**
   - 배포 항목 클릭 → "Build Logs" 확인
   - 에러 메시지 확인
